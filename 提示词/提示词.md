# SCM 波次自动推进 — 使用说明

## 为什么「触发器没自动继续」？

常见只有三类原因（可同时存在）：

### 1. Automation 没在 Cursor 里真正启用

仓库里的 `.cursor/automations/scm-wave-minute.prefill.json` **只是草稿**，不会自己跑。

你必须在 Cursor 里：

1. 打开 **Automations**（自动化）面板  
2. 新建或编辑名为 **「SCM 波次推进」** 的自动化  
3. 触发器选 **定时**，Cron 填 **每分钟**：`* * * * *`  
4. 工作区选 **开发-note**（整个笔记仓，不是只开 scm-platform 子文件夹）  
5. 把 `scm-wave-minute.instructions.txt` 里的说明贴进「指令」  
6. **打开开关 / 保存**

### 1.1 Cloud Agent 必须能识别 Git（你截图里的提示）

设置 → **Cloud Agents** 若显示「需要打开含 Git 的文件夹」：

1. **文件 → 打开文件夹**，选 **`e:\note\开发-note`**（整仓根目录，里面有 `.git`）
2. 左侧 **源代码管理** 能显示 Git 分支，即合格
3. 不要只开无 Git 的空目录；Automation 顶栏尽量别一直是 **No Repository**
4. **隐私模式**：不能用 Privacy Mode **(Legacy)**，需允许 Cloud Agent（见 Automations 右下角提示）
5. [Cloud Agents 仪表盘](https://cursor.com/dashboard?tab=cloud-agents) 确认已开启

### 2. 防重逻辑故意「跳过」

`progress.md` 里若 **W36 已全部打勾**，旧版指令会让 Agent 每分钟只写「跳过：当前波次已完成」就结束，看起来像没干活。

已改为：跟 **AGENTS.md 的「下一步」** 推进 **W37**，只有 2 分钟内重复空跑才跳过。

### 3. 和聊天里「接着做」不是一回事

- **Automation**：到点才启动**新的**云 Agent 会话  
- **本对话**：你说「继续」才会在当前会话里接着做  

要长期无人值守，靠第 1 步；要立刻推进，在本对话说「继续」即可。

## 本地替代：Loop（不依赖 Automation）

在 Agent 输入框：

```text
/loop 1m 读 scm-platform/AGENTS.md 和 progress.md，做 W37 一条，mvn test，更新 progress
```

会在本机每 1 分钟唤醒当前会话（需 Cursor 开着）。

## 相关文件

- 指令正文：`scm-platform` 同级的 `.cursor/automations/scm-wave-minute.instructions.txt`
- 进度防重：`scm-platform/progress.md`
- 波次目标：`scm-platform/AGENTS.md`
